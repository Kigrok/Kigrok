## [Nexul](https://github.com/Kigrok/Nexul) — Async Telegram Automation Framework

[![Python](https://img.shields.io/badge/Python-3.13-blue?logo=python)](https://www.python.org) [![Pyrogram](https://img.shields.io/badge/Pyrogram-2.0.106-orange?logo=Pyrogram)](https://docs.pyrogram.org/) [![aiofiles](https://img.shields.io/badge/aiofiles-24.1.0-blue?logo=aiofiles)](https://github.com/Tinche/aiofiles) [![cloudscraper](https://img.shields.io/badge/cloudscraper-1.2.71-orange?logo=cloudscraper)](https://github.com/venomous/cloudscraper) [![PyYAML](https://img.shields.io/badge/PyYAML-6.0.2-blue?logo=PyYAML)](https://github.com/yaml/pyyaml) [![MIT License](https://img.shields.io/badge/License-MIT-green)]()

Asynchronous multi-session automation client for Telegram mini-app ecosystems.

- Concurrent management of multiple Telegram sessions
- Automated reward claiming & task execution
- Human-like randomized intervals (anti-pattern avoidance)
- Proxy & user-agent configuration support
- YAML-based session configuration
- Event-driven async architecture (Pyrogram)

Designed as a scalable automation framework, not a single-purpose script.
